API Webserver: Applicant Tracking System

July 2023

Grade: 96% (High Distinction)

About

A few months after our first Python application in Term 1, my next coding assignment in Term 2 of Coder Academy involved creating an API webserver in Python, utilising Flask and our ORM of choice to solve a real-world problem. We had three weeks to select an app idea, submit a draft ERD for approval, develop and document the database and the APIs.

There was no specified number of API endpoints required, but it was suggested to have around four tables in our relational database and to ensure that each model included at least one of each of the four CRUD methods - Create, Read, Update, Delete. I opted to use SQLAlchemy as my ORM, given this was what was taught to us and it suited the app requirements.

I chose to base my relational database and webserver based on an application I am familiar with from my current occupation in HR - an Application Tracking System (ATS). Ultimately due to my familiarity with this type of system and desire to provide a good foundation for a future application, I ultimately ended up creating 7 tables and 35 endpoints.

Some of the key functionality of the webserver included:

  • encrypted user registration and login using Bcrypt
  • user session management and authentication using JWT
  • view open jobs
  • create, update and delete job listings (authorised users only)
  • update own user profile
  • update user permissions of other users (authorised users only)
  • apply for an open job
  • view candidates and applications (authorised users only)
  • update status on applications (authorised users only)
  • schedule interviews (authorised users only)
  • view your upcoming interviews, either as a candidate or interviewer
  • create interview scorecards (authorised users only)
